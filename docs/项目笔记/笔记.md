### GC
是什么
在单增的过程中是young gen生成的过程，那个断崖式下跌就是GC触发把所有young gen都清理掉了
为什么需要
堆内存是有限制的，满了就会卡住，所以需要把一些用不到的垃圾给清理掉
坑点
在GC过程中会触发STW，整个程序会停止执行，这个过程的快慢往往决定着程序运行速度

### exception 处理
是什么 
将具体是什么错误正确显示在控制台和客户端 
为什么需要 
把服务器内部异常翻译成客户端能理解的 HTTP 语义 
坑点
异常要继承 RuntimeException，否则每个方法都要显式声明 throws，代码很脏

### Virtual Threads



#### 自定义压测工具失败原因

HttpClient使用内部线程池处理请求，send()同步阻塞导致压测时线程行为被干扰，实际验证改用日志线程名对比。



#### 虚拟线程开启与否的对比

1.关闭虚拟线程时是用的线程池里的线程，
2.请求来了会创建虚拟线程，但虚拟线程跑在少量固定的carrier threads上，不是独占一个平台线程。
3.虚拟线程在等待时会从承载线程上下来，等到好了就会重新挂载到承载线程。而关闭时线程会一直在承载线程上面等着
